# Решение проблемы с командами /tell, /msg и т.д.

## Проблема
Команды `/tell`, `/msg`, `/pm`, `/w` и подобные не работают, выдается ошибка "Test ActionBar" с сообщением о неправильных аргументах. Это происходит из-за конфликта регистрации команд - другой плагин или система Velocity регистрирует эти команды первыми.

## Что было сделано

### 1. Добавлена защита от конфликтов в CommandManager
- Добавлена попытка отмены регистрации существующих команд перед регистрацией новых
- Добавлено логирование успешных и неудачных регистраций команд
- При неудаче регистрации команды, пользователю предлагается альтернатива

### 2. Создана мета-команда /chat
Добавлен новый класс `ChatCommand.java`, который предоставляет единую точку входа для всех chat-функций:

**Использование:**
- `/chat msg <игрок> <сообщение>` - отправить приватное сообщение
- `/chat tell <игрок> <сообщение>` - отправить приватное сообщение
- `/chat pm <игрок> <сообщение>` - отправить приватное сообщение
- `/chat w <игрок> <сообщение>` - отправить приватное сообщение
- `/chat reload` - перезагрузить конфигурацию (требует права `chat.reload`)

## Как использовать

### Если обычные команды работают
Продолжайте использовать `/msg`, `/tell`, `/pm`, `/w`, `/message` как обычно.

### Если обычные команды не работают
Используйте команду `/chat` с подкомандами:
- Вместо `/tell Player Hello` используйте `/chat tell Player Hello`
- Вместо `/msg Player Hi there` используйте `/chat msg Player Hi there`

### Проверка статуса команд
1. Запустите сервер с обновленным плагином
2. Проверьте логи - должны появиться сообщения о регистрации команд:
   ```
   [INFO]: Successfully registered private message command: /msg
   [INFO]: Successfully registered private message command: /tell
   ```
   или
   ```
   [ERROR]: Failed to register command 'tell': <причина>. You can use '/chat tell' instead.
   ```

## Настройки

Все настройки находятся в `config.toml`:

```toml
[private_messages]
# Включить приватные сообщения
enable = true

# Псевдонимы команд для приватных сообщений
aliases = ["msg", "tell", "pm", "w", "message"]

# Другие настройки...
```

## Дополнительные функции

### Автодополнение
- `/chat` покажет все доступные подкоманды
- `/chat msg <TAB>` покажет список онлайн игроков

### Звуковые уведомления
При получении приватного сообщения игрок услышит звук (настраивается в `config.toml`).

### Логирование
Все приватные сообщения могут логироваться в консоль (настройка `log_to_console`).

## Устранение неполадок

### Если команды все еще не работают
1. Проверьте права доступа - убедитесь, что у игроков есть право `chat.msg`
2. Проверьте, что `private_messages.enable = true` в конфигурации
3. Используйте команду `/chat reload` для перезагрузки конфигурации
4. В крайнем случае, всегда можно использовать `/chat <подкоманда>`

### Если нужно добавить новые псевдонимы
Добавьте их в массив `aliases` в секции `[private_messages]` файла `config.toml`:
```toml
aliases = ["msg", "tell", "pm", "w", "message", "whisper", "dm"]
```

## Преимущества решения
1. **Обратная совместимость** - старые команды работают, если нет конфликтов
2. **Резервный механизм** - команда `/chat` всегда доступна
3. **Улучшенное логирование** - понятно, какие команды зарегистрированы
4. **Гибкость** - легко добавлять новые псевдонимы
5. **Диагностика** - ясные сообщения об ошибках с инструкциями по использованию
